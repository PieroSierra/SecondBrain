# Implementation Plan: Personal Knowledge Base (Second Brain)

**Branch**: `001-personal-knowledge-base` | **Date**: 2026-06-16 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/001-personal-knowledge-base/spec.md`

## Summary

Build a self-organising personal knowledge base driven entirely by Claude Code skills. The user drops content into `raw/`, runs an ingest skill that synthesises the material into a cross-linked `wiki/`, and queries the wiki with natural language. Dedicated import skills handle Craft notes (via MCP) and PDFs. A setup skill bootstraps the vault by generating `CLAUDE.md` with the user's declared interests. All state is stored in local markdown files and a lightweight JSON manifest — no external services required.

## Technical Context

**Language/Version**: No traditional programming language — the system is implemented entirely as Claude Code skill instruction files (markdown) with no compiled code  
**Primary Dependencies**: Claude Code CLI, Craft MCP integration (pre-configured per spec assumption), Claude's native PDF reading capability  
**Storage**: Local filesystem — markdown files for all content; `raw/.ingest-manifest.json` for ingestion state  
**Testing**: Manual acceptance testing via skill invocations against known fixtures; no automated test framework  
**Target Platform**: macOS (Claude Code CLI, current working directory)  
**Project Type**: Skill-based knowledge management system (Claude Code skills)  
**Performance Goals**: Ingest batch of ≤20 files in <2 minutes; queries answered in <60 seconds  
**Constraints**: No external services or third-party runtime dependencies; ingest treats `raw/` as read-only while users and importers may update sources
**Scale/Scope**: Single user, personal corpus; expected to grow to hundreds of wiki articles over months
